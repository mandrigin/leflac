# LE FLAC 1.1.0 — Android Auto car deck

LF-1 is now a projected Android Auto media source while preserving the existing
phone faceplate and offline-only design.

## Highlights

- Browsable Hot now, Albums, Mixes, and All songs shelves.
- Playback, queue resume, steering-wheel transport controls, search, and voice
  query resolution through one Media3 library session.
- New independent `CAR SKIN` setting: POCKET is the default and FIELD is the
  alternate. The phone skin setting is unchanged.
- Permission-safe cold start and a parked setup message when local-audio access
  has not yet been granted on the phone.
- Local artwork only; the app still requests no Internet permission.

See the [Android Auto operator guide](https://github.com/mandrigin/leflac/blob/v1.1.0/docs/ANDROID_AUTO.md)
for Honda e setup,
first-drive checks, and troubleshooting. Phone-side library integration is
verified. Physical Honda e head-unit, voice, steering-wheel, audio-focus,
day/night, and rotary acceptance remain live operator checks; Android Auto owns
the final car-screen chrome.

## Tested build

- App version: `1.1.0` (`versionCode 2`)
- Platform: Android 14+; projected Android Auto (not Android Automotive OS)
- Device: Nothing A024, Android 16 / API 36
- Passed: unit tests, Android lint, debug assembly, Android Auto cold-browser
  instrumentation, and the existing on-device FFT benchmark
- Upgrade-installed and cold-launched without clearing the existing app data or
  local-audio permission

The attached `LE-FLAC-1.1.0-Android-Auto-debug.apk` is the exact tested APK. It
is development/debug signed for direct sideload testing and is not a Play Store
