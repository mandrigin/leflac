# LE FLAC 1.3.0 — UX consistency pass

LF-1 keeps the same FIELD/POCKET faceplate and playback model, but its controls
now tell the truth more consistently across lifecycle changes, rotation,
accessibility services, and projected Android Auto.

## Highlights

- Returning from system Settings immediately rechecks Music and audio access,
  refreshes the local library, and removes the stale `MUSIC ACCESS REQUIRED`
  state without restarting the app.
- System Back unwinds the visible state first: rear manual, Up Next selection,
  search, and open album before leaving LF-1.
- Ordered Up Next selection survives rotation, reconciles MediaStore rescans,
  and reports why a current/already-queued/wrong-pool track cannot be held.
- HOT, MIXES, ALBUMS, NEXT, and search retain their browse positions. Queue
  confirmation reports how many tracks were actually inserted.
- Empty-deck, startup mix, long-title, scrubber, and previous/next punch-in
  states are bounded and symmetric instead of showing stale or impossible
  machine status.
- Album headers, title badges, durations, service information, the custom
  scrubber rail/readout, disabled PLAY face, and strip clock are registered to
  the existing chassis grid without changing the faceplate geometry.
- Transport, rail, rear switches, search, albums, Up Next actions, and service
  controls now expose consistent roles, labels, selected/toggle state, disabled
  state, and practical touch targets while retaining the existing hardware art.

## Android Auto clarification

`CAR SKIN` is now `AA ART`, because Android Auto owns its dashboard chrome,
typography, and standard transport buttons. The switch changes LE FLAC's
POCKET/FIELD artwork and supported content-style hints; it cannot project the
phone's Compose faceplate.

An ADB-installed media app does not need a public Play listing, but Android Auto
developer mode and **Unknown sources** must be enabled while parked. Also enable
**LE FLAC** under **Customize launcher**, disconnect/reconnect, and launch it
from the Android Auto app drawer/media-source picker. The generic active-media
card is only the shared MediaSession surface and can correctly redirect phone
Activity work back to the parked phone.
