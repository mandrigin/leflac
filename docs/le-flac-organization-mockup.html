ORGANIZATION MOCKUP

This page draws the proposed grammar: strict zones, one dominant element per state, clearer transport semantics, and a calmer ledger below the expressive faceplate.

FOUR ZONES

FACEPLATENow playing, analysis state, no Toy preview.
TRANSPORTScrubber, time, immediate controls.
LEDGERLibrary, mixes, albums, SCAN.
BACK PANELManual, settings, DIP switches, serial.

CONTROL GRAMMAR

Square buttons = immediate transport
ORDER rail = deterministic next/previous
RNG rail = smart generated future
Rail latch = ORDER or RNG future
Hold PREV/NEXT = force ORDER override
DIP switches = persistent settings
Ledger rows = music selection
SCAN = command or filter
Glyph = system-strip state only
Orange = active/current/action
Cyan = RNG/analysis/deck signal

COMPOSITION RULES

Four stacked plates, no floating widgets
Faceplate = largest quiet field
Transport = compressed hardware deck
Ledger = dense but calm reading surface
Latch spans NEXT and RNG because it reroutes future
Buttons never fade to explain state
State lives in labels, latches, LEDs, and rails

COLOR RULES

Orange = current/action/hot/selected
Ink = ORDER rail and readable content
Cyan = RNG rail, analysis, deck signal
Dim = engraved secondary structure
FIELD allows orange + cyan contrast
POCKET collapses all semantics into LCD ink and tone

WHAT CHANGED

The top strip exposes mode and a quiet Glyph-ready state. The now-playing area has one hero: the music. The scrubber and controls share a transport deck. The library is a readable ledger instead of a debug surface. FIELD and POCKET keep the same layout but use different material rules.

CHANGE LIST

  • Remove Toy from the front. Glyph is no longer a visual card or hero element on the playback surface.
  • Keep Glyph as quiet system state. The front only says `GLYPH READY`; full Toy behavior belongs to hardware, rear panel, or settings.
  • Make music the Faceplate hero. Title, artist, analysis state, and playback context own the top area.
  • Rename Rotation to Hot Now. The library shelf should say what it means: current heat, not an abstract system term.
  • Use a strict vertical stack. System Strip, Faceplate, Transport Deck, Ledger.
  • Move telemetry into a compact strip. `K`, `B`, `S`, cue count, analyzed state, or mix map appear as low-volume machine data.
  • Use two-rail transport. `NEXT` follows ORDER when AUTO is off and the RNG queue when AUTO is on.
  • Keep RNG as its own control. `RNG` is a one-shot random jump when AUTO is off; with AUTO on it advances or refreshes the RNG future.
  • Use a rail latch, not transparency. Buttons stay solid; ORDER is ink, RNG is cyan, and the selector shows where NEXT will go.
  • Use hold as override. Holding PREV/NEXT forces the ORDER rail even when AUTO/RNG is active.
  • Keep command syntax inside SCAN. `.hot`, `.gym`, `.mix`, `.rng` stay in the prompt, not on hardware buttons.
  • Make the Ledger readable. Rows prioritize number, title, artist/context, duration, and selected/hot state.
  • Move file trivia out of default rows. Bitrate, size, plays, skips, and deep analysis go to service mode or rear panel.
  • Support long titles. Five-word titles step down in size, use balanced wrapping, and keep the controls stable.
  • One layout, two materials. FIELD and POCKET share structure; FIELD uses chassis/orange/cyan, POCKET uses LCD ink and solid tones.
  • One thing loud at a time. The current track can be expressive; the library and controls stay calmer.
  • Use color semantically. Orange means current/action/hot. Ink means ORDER. Cyan means RNG/analysis/deck signal. Dim means secondary but readable.
  • Preserve the back panel. Manual, DIP switches, serial, runtime, command legend, and advanced settings live on the rear.
